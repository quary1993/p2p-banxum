# Admin TODO Index

Status: Split into audience-specific TODO files.
Last updated: 2026-05-30.

The original `ADMIN_TODO.md` has been split into three files so each audience sees the decisions and inputs relevant to them.

Use these files going forward:

- `admin_todo_accounts.md`: third-party accounts, credentials, API access, domains, bank portals, cloud accounts, and other access that Garanta or the tech team must provide.
- `admin_todo_garanta.md`: legal, procedural, accounting, finance, product, operational, and compliance decisions. This file is written for non-technical Garanta admins and advisors.
- `admin_todo_tech.md`: architecture and strictly technical implementation decisions owned by the technical team.

Each file separates:

- Blocking items: needed before a specific module can be completed beyond generic scaffolding or mocks.
- Non-blocking items: not required to continue implementation, but needed before production launch, later configuration, operational polish, or future releases.
